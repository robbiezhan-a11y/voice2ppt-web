# PPT 语音 AI 改写工具 开发技术文档（V1.1）

## 1、项目概述

本工具基于 Python 桌面开发，实现 **麦克风实时录音 → OpenAI Whisper 语音识别 → GPT 智能文案改写 → 一键批量写入选中 PPT 幻灯片**，适配 Office PowerPoint、WPS 演示，全程可视化 GUI 操作。

V1.1 相对原版修复了 5 处致命 Bug 并完成 13 项健壮性改进，详见 `CHANGELOG.md`。

## 3、模块分工详解

### 3.1 配置模块 `config/config.json`

存储字段：
| 字段 | 类型 | 说明 |
|---|---|---|
| `api_key` | string | **AES-256-CBC 加密后**的 OpenAI 密钥（Base64 编码） |
| `model_name` | string | AI 模型名称（gpt-3.5-turbo / gpt-4 / gpt-4o / gpt-4o-mini） |
| `last_check_time` | int | 上次密钥校验时间戳 |
| `base_url` | string | 接口地址（默认 `https://api.openai.com`，支持中转） |

加密方案：
- 算法：AES-256-CBC
- 密钥：`sha256("PPTVoiceRewrite_v1.1_local_secret_key")` → 32 字节
- IV：每次加密随机 16 字节
- 存储：`base64(IV + ciphertext)`

相关函数：`encrypt_key(plaintext)` / `decrypt_key(blob)`

### 3.2 音频录制模块

- 采样率：16000 Hz（Whisper 最优识别采样率）
- 声道：单声道（减小体积、提升识别速度）
- 采样位数：16 bit
- 录制方式：`RecordThread(QThread)` 后台线程循环读取麦克风数据流
- VU 电平：每 chunk 计算 RMS 归一化到 0-100，通过 `vu_signal` 信号驱动电平条
- 录音时长：`QTimer` 每秒更新 `duration_label`
- 资源释放：`_cleanup()` 确保 `stop_stream / close / wf.close / pyaudio.terminate` 全链路执行
- 异常处理：`stream.read(exception_on_overflow=False)`，停止瞬间不再抛 `OSError`

**重要改进**：每次录音创建独立 `pyaudio.PyAudio()` 实例，避免原版全局实例不释放导致的设备占用。

### 3.3 AI 接口模块

#### 3.3.1 密钥校验（轻量化）
- 接口：`GET {base_url}/v1/models`
- 用途：启动时后台校验密钥有效性
- 优点：不消耗 token（原版用 chat completion 校验每次扣费）

#### 3.3.2 语音转写接口
- 接口：`POST {base_url}/v1/audio/transcriptions`
- 提交 wav 音频文件，返回纯文本口述内容
- **前置校验**：文件大小 ≤ 25MB（Whisper 限制），超限提示用户缩短录音
- 超时：120 秒（长录音上传需要时间）

#### 3.3.3 对话改写接口
- 接口：`POST {base_url}/v1/chat/completions`
- system 角色：用户自定义改写要求
- user 角色：识别出的语音原文
- 返回：优化后文案
- 超时：90 秒
- temperature：0.65

#### 3.3.4 状态码检查（V1.1 新增）
所有 API 响应先检查 `status_code != 200`，提取错误信息：
- 401 → 密钥无效
- 429 → 额度不足
- 5xx → 服务端异常
- 其他 → 显示原始错误文本（截断 200 字符）

#### 3.3.5 取消机制（V1.1 新增）
- `AIProcessThread` 持有 `requests.Session`
- 用户点击「取消 AI 处理」→ `cancel()` 设置标志 + `session.close()` 中断请求
- 转写与改写两步之间检查 `_cancel` 标志，及时退出

### 3.4 PPT 写入模块

#### 3.4.1 实例绑定（V1.1 修复）
```python
# 优先绑定已运行的 PowerPoint 实例（避免新开进程）
try:
    ppt_app = win32com.client.GetActiveObject("PowerPoint.Application")
except Exception:
    ppt_app = win32com.client.Dispatch("PowerPoint.Application")
```

#### 3.4.2 前置检查（V1.1 新增）
四道检查，任一失败给出明确提示：
1. `ppt_app.Presentations.Count == 0` → "未检测到已打开的 PPT 文件"
2. `ppt_app.ActiveWindow is None` → "未找到活动 PPT 窗口"
3. `sel.SlideRange.Count == 0` → "未选中任何幻灯片"
4. 无可写入文本框 → "选中幻灯片内没有可写入的文本框"

#### 3.4.3 分段写入（V1.1 核心改进）
原版缺陷：所有文本框写入相同 content → 10 个框变成同一句话。

V1.1 改进：
1. 收集选中幻灯片内所有 `HasTextFrame == -1 && HasText == -1` 的文本框（按幻灯片顺序）
2. 改写结果按 `\n\s*\n`（空行）切分为段落
3. 依次写入不同文本框：`boxes[i].TextRange.Text = paragraphs[i]`
4. 段落数 > 文本框数：余下段落追加到最后一个文本框
5. 段落数 < 文本框数：多余文本框保持不变（不清空）

#### 3.4.4 原文备份（V1.1 新增）
写入前读取每个文本框原始内容，存入历史记录 JSON：
```json
{
  "backup": [
    {"slide": 1, "box_index": 0, "original": "原标题"},
    {"slide": 1, "box_index": 1, "original": "原正文"}
  ]
}
```

⚠️ 限制：必须提前手动打开 PPT 文件并选中幻灯片，软件不会自动打开 PPT 文件。

### 3.5 历史记录模块

每次完成改写自动生成 JSON 存入 `history/` 目录：
| 字段 | 说明 |
|---|---|
| `time` | 操作时间 |
| `origin` | 语音识别原文 |
| `rewrite` | AI 改写结果（用户可能编辑过） |
| `slide_count` | 修改的幻灯片页数 |
| `box_count` | 选中文本框总数 |
| `written` | 实际写入的文本框数 |
| `backup` | 写入前各文本框原始内容（用于回滚） |

历史记录弹窗支持左侧列表 + 右侧详情查看，含原文备份展示。

### 3.6 密钥校验模块

程序启动后台异步线程联网检测密钥有效性：
- `KeyCheckThread(QThread)` 调用 `/v1/models` 轻量接口
- 四种状态提示：
  - ✅ 密钥校验正常（200）
  - ❌ 密钥无效（401）
  - ⚠️ API 调用次数超限/余额不足（429）
  - ⚠️ 联网校验失败（网络异常）

## 4、线程设计说明

| 线程类 | 用途 | 信号 |
|---|---|---|
| `RecordThread` | 录音循环 | `vu_signal(int)` / `finished_signal(str)` / `error_signal(str)` |
| `KeyCheckThread` | 密钥校验 | `finish_signal(str)` / `error_signal(str)` |
| `AIProcessThread` | 语音转写+改写 | `finished_signal(str,str)` / `error_signal(str)` / `progress_signal(str)` |

设计要点：
1. 所有网络请求、录音循环、AI 耗时操作全部使用独立 QThread，禁止主线程阻塞
2. Qt 禁止跨线程直接操作 UI 控件，使用信号槽传递结果
3. COM 组件调用前后执行 `CoInitialize/CoUninitialize`，规避多线程 COM 崩溃
4. `MainWin.closeEvent` 统一清理：停止录音、取消 AI、等待线程退出（最多 3 秒）

## 5、路径设计说明（V1.1 修复）

```python
# 用户数据路径（exe 同级，可持久化）
if getattr(sys, 'frozen', False):
    BASE_PATH = os.path.dirname(sys.executable)  # 打包后：exe 目录
else:
    BASE_PATH = os.path.dirname(os.path.abspath(__file__))  # 开发态：脚本目录

# 资源路径（打包后走 _MEIPASS 临时解压目录）
def resource_path(relative):
    if getattr(sys, 'frozen', False):
        base = getattr(sys, '_MEIPASS', os.path.dirname(sys.executable))
    else:
        base = os.path.dirname(os.path.abspath(__file__))
    return os.path.join(base, relative)
```

| 路径 | 用途 | 是否持久化 |
|---|---|---|
| `BASE_PATH/config/` | 配置文件 | ✅ 持久化 |
| `BASE_PATH/record/` | 录音文件 | ✅ 持久化 |
| `BASE_PATH/history/` | 历史记录 | ✅ 持久化 |
| `resource_path("icon.ico")` | 图标 | ❌ 打包进 exe |

## 6、已知限制 & 注意事项

1. 仅支持 Windows 系统（依赖 win32com），macOS/Linux 无法操控 PPT
2. 需联网环境访问 OpenAI 接口（或自定义中转 `base_url`）
3. API 密钥需自行在 OpenAI 平台申请充值
4. WPS 需完整安装版，精简绿色版缺 COM 组件
5. 录音文件不会自动清理，定期手动删除 `record/` 冗余 wav
6. Whisper 单文件 25MB 限制（约 15 分钟录音），超限会提示
7. **VC++ 2015-2022 x64 运行库**为前置依赖，缺失会闪退
