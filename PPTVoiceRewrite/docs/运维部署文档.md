# PPT 语音 AI 改写工具 运维 & 部署维护文档（V1.1）

## 1、首次部署流程

### 1.1 开发部署（源码运行）

1. 解压项目源码压缩包
2. 安装 Python 3.10 环境（勾选 Add Python to PATH）
3. **安装 VC++ 2015-2022 x64 运行库**（PyQt6/pywin32 依赖）
   - 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe
4. CMD 进入项目目录执行依赖安装：
   ```cmd
   pip install -r requirements.txt
   ```
5. 运行调试：
   ```cmd
   python main.py
   ```
6. 测试录音、识别、改写、PPT 写入全部流程无误后打包

### 1.2 用户终端部署（成品交付）

#### 方式 A【便携绿色版】
1. 将 `dist\PPTVoiceRewrite\` 整个文件夹复制到电脑任意位置
2. 确认已安装 VC++ 2015-2022 x64 运行库
3. 双击 `PPTVoiceRewrite.exe` 直接使用，无需安装
4. **无需手动创建 config/record/history 文件夹**（程序自动创建）

#### 方式 B【安装版】
1. 双击 `PPTVoiceRewrite_Setup_v1.1.exe` 按向导下一步完成安装
2. 桌面自动生成快捷方式，点击启动
3. 升级时直接覆盖安装，config/record/history 不丢失

> ⚠️ 两种方式都**必须安装 VC++ 2015-2022 x64 运行库**，否则闪退。

## 2、日常使用运维要点

### 2.1 密钥维护
- OpenAI 密钥存在有效期、额度限制
- 提示「密钥无效」/「额度不足」时需更换充值后的新 key
- **V1.1 密钥已加密存储**（AES-256-CBC），但仍属私密凭证，不要泄露 config.json
- 支持自定义 `base_url`，可配置中转 API 地址

### 2.2 文件清理运维
| 目录 | 内容 | 清理建议 |
|---|---|---|
| `record/` | 录音 wav 文件 | 每月定期清理过期录音 |
| `history/` | 历史记录 JSON | 按需保留，可直接删除 |
| `config/` | 配置文件 | **切勿删除**（含加密密钥） |

### 2.3 软件异常运维

| 现象 | 排查方案 |
|---|---|
| 程序启动空白闪退 | 1. 安装 VC++ 2015-2022 x64 运行库<br>2. 删除 `config/config.json` 重置配置重新打开<br>3. 用控制台模式启动查看报错 |
| 无法写入 PPT | 1. 关闭所有 PPT/WPS 重新打开<br>2. 确认已选中幻灯片（非空白处）<br>3. WPS：设置 → 高级 → 启用 COM 自动化<br>4. 右键以管理员身份运行 |
| 录音无声音 | 1. 电脑麦克风权限开启<br>2. 切换默认录音设备<br>3. 观察电平条是否有波动 |
| 识别内容为空 | 1. 网络不通<br>2. 密钥失效<br>3. 环境嘈杂录音模糊<br>4. 录音过短（< 1 秒） |
| API 请求超时 | 1. 检查网络能否访问 OpenAI 接口<br>2. 尝试配置中转 base_url<br>3. 长录音上传需较长时间（已设 120s 超时） |
| Whisper 文件过大 | 录音超 25MB 会提示，缩短录音时长（约 15 分钟以内） |
| AI 处理卡住 | 点击「⏹ 取消 AI 处理」按钮中断，重新录音 |

## 3、权限与环境运维

### 3.1 Windows 系统权限
- 软件右键【以管理员身份运行】避免 COM 权限不足
- 安装版默认需管理员权限（`PrivilegesRequired=admin`）

### 3.2 WPS 办公软件
- **不能使用精简绿化版**，必须完整官方安装版
- WPS 设置 → 高级设置 → 勾选「启用 COM 自动化」

### 3.3 麦克风权限
- Windows 设置 → 隐私和安全性 → 麦克风
- 开启「桌面应用」麦克风权限
- 确认本工具被允许使用麦克风

### 3.4 VC++ 运行库（V1.1 新增重点）
- **必须安装 VC++ 2015-2022 x64**
- 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe
- 缺失症状：双击 exe 无反应 / 闪退 / 报错 0xc000007b
- 验证方法：控制面板 → 程序和功能 → 查找 "Microsoft Visual C++ 2015-2022 Redistributable (x64)"

### 3.5 网络环境
- 需访问 `api.openai.com`（或自定义 base_url）
- 国内网络可能需要代理或中转 API
- 配置路径：密钥设置 → 接口地址

## 4、版本更新运维

### 4.1 源码更新
1. 替换新版 `main.py`
2. 重新执行 `build.bat` 打包
3. 用 Inno Setup 重新编译 `install.iss`

### 4.2 成品 exe 更新（便携版）
1. 直接替换 `PPTVoiceRewrite.exe` 文件
2. **保留 config 文件夹**，密钥不丢失
3. record/history 数据保留

### 4.3 安装版更新
1. 新版本直接覆盖安装
2. 配置文件不会被覆盖删除（`onlyifdoesntexist` 标志）
3. record/history 目录永不卸载（`uninsneveruninstall` 标志）

### 4.4 配置迁移
- 升级前备份 `config/config.json`
- 升级后如配置丢失，恢复 config.json 即可
- 密钥加密格式向下兼容（V1.1 加密，V1.0 明文无法读取）

## 5、故障排查清单

| 故障现象 | 排查方案 |
|---|---|
| 点击录音无反应 | 麦克风硬件 / 系统权限 / pyaudio 运行异常 / 查看电平条 |
| 录音电平条无波动 | 麦克风未授权 / 默认设备错误 / 麦克风被占用 |
| 识别内容为空 | 网络不通 / 密钥失效 / 环境嘈杂 / 录音过短 |
| AI 改写失败 HTTP 401 | 密钥无效，重新配置 |
| AI 改写失败 HTTP 429 | 额度不足，充值或换 key |
| AI 改写失败 HTTP 5xx | OpenAI 服务异常，稍后重试 |
| 无法修改 PPT 内容 | PPT 未选中页面 / WPS 缺 COM 组件 / PPT 只读模式 |
| PPT 写入提示"无可写入文本框" | 选中的幻灯片内文本框为空或无文本框 |
| 启动程序报错闪退 | 配置文件损坏 / 缺 VC++ 运行库 / 缺 pywin32 |
| 校验密钥一直失败 | 网络无法访问外网 / API 接口地址受限 / base_url 配置错误 |
| 写入后所有文本框内容相同 | 已是 V1.0 已知问题，V1.1 已修复（分段写入），确认运行的是 V1.1 |
| 历史记录丢失 | record/history 在 exe 同级目录，确认未误删 |

## 6、安全运维说明

1. **API 密钥属于私密凭证**，不要随意泄露、转发
2. V1.1 密钥已 AES-256-CBC 加密存储，但仍禁止把 `config.json` 打包分发
3. 软件无后台上传行为，所有录音 / 历史 / 配置仅本地存储
4. 不联网情况下仅录音可用，AI 识别改写完全失效
5. 历史记录含 PPT 原文备份，含敏感内容时注意物理安全
6. 卸载软件不会删除 record/history/config（`uninsneveruninstall`），需手动清理

## 7、数据备份与恢复

### 7.1 备份
- 关键目录：`config/`、`history/`
- 复制整个文件夹到备份位置即可
- `record/` 可选备份（录音文件较大）

### 7.2 恢复
- 将备份的 `config/`、`history/` 覆盖回安装目录
- 重启程序自动读取

### 7.3 PPT 原文回滚
V1.1 历史记录含 `backup` 字段，记录写入前各文本框原文：
```json
{
  "backup": [
    {"slide": 1, "box_index": 0, "original": "原标题"}
  ]
}
```
如需回滚，可手动打开历史 JSON，将 `original` 内容复制回 PPT 对应文本框。

## 8、性能与容量

| 资源 | 单次占用 | 累积建议 |
|---|---|---|
| 录音文件 | 约 1.8MB/分钟（16K 单声道 16bit） | 每月清理 |
| 历史记录 | 约 2-10KB/次 | 可长期保留 |
| 配置文件 | < 1KB | 永久保留 |
| Whisper 上传 | ≤ 25MB/次 | - |
| 内存占用 | 约 80-120MB | - |
| 打包后体积 | 约 60-80MB（含 PyQt6） | 可用 upx 压缩 |
